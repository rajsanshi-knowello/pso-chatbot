# PSO Review Service

FastAPI microservice for editorial review of PSO documents. n8n calls this service over HTTP; it returns structured JSON covering 10 editorial categories from the PSO Style Guide.

---

## Local setup

**Requirements:** Python 3.11+

```bash
# 1. Clone / navigate to the project root
cd PSO_Review_Service

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure environment
cp .env.example .env
# Edit .env and set API_KEY to any strong random string, e.g.:
# API_KEY=supersecretlocalkey

# 5. Start the server
uvicorn app.main:app --reload
# → Server running at http://localhost:8000
# → Logs output as JSON to stdout
```

## Deploying to Render

1. Push the repository to GitHub (the `Dockerfile` and `render.yaml` are already configured).
2. In Render, click **New → Blueprint** and connect the repository.
3. Render reads `render.yaml` and creates a **free-tier Web Service** named `pso-review-service`.
4. `API_KEY` is auto-generated by Render on first deploy — copy it from the **Environment** tab.
5. The service is live once the health check at `/health` returns 200.

To redeploy: push a new commit or click **Manual Deploy** in the Render dashboard.

## How n8n calls this service

Add an **HTTP Request** node in your n8n workflow:

| Field | Value |
|---|---|
| Method | POST |
| URL | `https://<your-render-url>/review` |
| Authentication | Header Auth |
| Header name | `X-API-Key` |
| Header value | *(the API_KEY from Render's Environment tab)* |
| Body content type | JSON |
| Body | `{ "document_url": "{{ $json.document_url }}", "session_id": "{{ $json.session_id }}" }` |

For the chat endpoint, change the URL to `/chat` and the body to:

```json
{ "message": "{{ $json.message }}", "session_id": "{{ $json.session_id }}" }
```

## Sample curl commands

### Health check (no auth required)
```bash
curl http://localhost:8000/health
# {"status":"ok"}
```

### POST /review
```bash
curl -s -X POST http://localhost:8000/review \
  -H "X-API-Key: supersecretlocalkey" \
  -H "Content-Type: application/json" \
  -d '{"document_url": "https://example.com/policy.docx", "session_id": "abc123"}' \
  | python -m json.tool
```

### POST /chat
```bash
curl -s -X POST http://localhost:8000/chat \
  -H "X-API-Key: supersecretlocalkey" \
  -H "Content-Type: application/json" \
  -d '{"message": "What should I fix first?", "session_id": "abc123"}' \
  | python -m json.tool
```

### POST /review/upload (multipart — for n8n binary uploads)

**Linux/Mac:**
```bash
curl -s -X POST http://localhost:8000/review/upload \
  -H "X-API-Key: supersecretlocalkey" \
  -F "file=@/path/to/your/document.docx" \
  -F "session_id=abc123" \
  | python -m json.tool
```

**Windows PowerShell:**
```powershell
curl.exe -s -X POST http://localhost:8000/review/upload `
  -H "X-API-Key: supersecretlocalkey" `
  -F "file=@C:\path\to\document.docx" `
  -F "session_id=abc123"
```

**n8n HTTP Request node (multipart):**

| Field | Value |
|---|---|
| Method | POST |
| URL | `https://<your-render-url>/review/upload` |
| Authentication | Header Auth → `X-API-Key` |
| Body content type | Form-Data/Multipart |
| Fields | `file` = Binary input, `session_id` = `{{ $json.session_id }}` |
